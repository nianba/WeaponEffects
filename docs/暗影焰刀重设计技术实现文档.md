# 暗影焰刀重设计技术实现文档：影焰回刃

本文档面向当前 `WeaponEffects` 项目实现。建议 **不要把暗影焰刀塞进现有剑气连段系统** ，而是做成一个独立的武器特例：左键发射自定义高速飞刀，命中后生成驻留刀；右键扫描并召回驻留刀。这样改动范围小，不需要碰现有 `SlashArcProjectile` 的复杂 Shader / 三角带绘制流程。

当前项目的架构本来就是用 `GlobalItem` 作为武器行为入口，`Content/Projectiles` 放战斗投射物，`Common/Players` 放玩家状态，`Common/Configs` 放玩法/视觉配置；这次实现应沿用这个组织方式。([GitHub](https://raw.githubusercontent.com/nianba/WeaponEffects/master/ARCHITECTURE.md "raw.githubusercontent.com")) tModLoader 的 `GlobalItem.AltFunctionUse` 可让物品支持右键，并在右键使用时将 `player.altFunctionUse` 设为 `2`；`GlobalItem.Shoot` 返回 `false` 可阻止原版射击逻辑继续执行，适合右键召回这种自定义行为。([tModLoader 文档](https://docs.tmodloader.net/docs/1.4-stable/class_terraria_1_1_mod_loader_1_1_global_item.html "tModLoader: Terraria.ModLoader.GlobalItem Class Reference"))

---

## 1. 实现原则

本版目标是“好实现、好调试、风险低”，因此不做以下复杂方案：

```text
不改现有 SlashArcProjectile 剑气绘制
不接入现有普通攻击四段连段
不做复杂 ribbon / mesh / shader 刀光
不做复杂路径预测
不做自定义网络包
不做驻留刀跟随敌人
不做召回刀曲线回收
```

首版只用：

```text
GlobalItem 特例分流
3～4 个简单 ModProjectile
WeaponEffectsPlayer 记录召回命中计数
原版暗影焰刀贴图
少量 Dust / 已有命中特效
```

核心思路是：

```text
左键：原武器改为发射 ShadowFlamePierceKnifeProjectile
命中：OnHitNPC 生成 ShadowFlameStuckKnifeProjectile
右键：扫描玩家自己的驻留刀，生成 ShadowFlameRecallKnifeProjectile
召回命中：WeaponEffectsPlayer 统计同一次召回内 NPC 被命中的刀数
达到 4 把：生成 ShadowFlameExplosionProjectile
```

## 4. 与现有挥砍系统隔离

### 4.1 修改 `SlashGlobalItem.ShouldUseSlashAction`

必须显式排除暗影焰刀，避免被现有近战重做系统接管。

在 `ShouldUseSlashAction(Item item)` 前半段加入：

```csharp
if (item.type == ItemID.ShadowFlameKnife)
{
    return false;
}
```

这样暗影焰刀完全走新分支，不进入普通剑气、蓄力斩、连段逻辑。

### 4.2 新增 `ShadowFlameKnifeGlobalItem`

文件：

```text
Content/Items/ShadowFlameKnifeGlobalItem.cs
```

职责：

```text
只处理 ItemID.ShadowFlameKnife
左键发射自定义穿刺飞刀
右键召回驻留刀
阻止右键时原版射击
```

`tModLoader.ModProjectile` 的 `AI`、`OnHitNPC`、`OnTileCollide` 等 Hook 正适合实现自定义投射物移动、命中和撞墙逻辑；额外字段若要同步，应使用 `SendExtraAI` / `ReceiveExtraAI`。([tModLoader 文档](https://docs.tmodloader.net/docs/stable/class_mod_projectile.html "tModLoader: ModProjectile Class Reference"))

核心规则：

```csharp
public override bool AppliesToEntity(Item entity, bool lateInstantiation)
{
    return entity.type == ItemID.ShadowFlameKnife;
}

public override void SetDefaults(Item item)
{
    item.shoot = ModContent.ProjectileType<ShadowFlamePierceKnifeProjectile>();
    item.shootSpeed = ShadowFlameKnifeTuning.PierceSpeed;
    item.noMelee = true;
    item.noUseGraphic = true;
}

public override bool AltFunctionUse(Item item, Player player)
{
    return item.type == ItemID.ShadowFlameKnife;
}

public override bool Shoot(
    Item item,
    Player player,
    EntitySource_ItemUse_WithAmmo source,
    Vector2 position,
    Vector2 velocity,
    int type,
    int damage,
    float knockback)
{
    if (player.altFunctionUse == 2)
    {
        if (player.whoAmI == Main.myPlayer)
        {
            ShadowFlameKnifeHelper.RecallAll(player, source, damage, knockback);
        }

        return false;
    }

    return true;
}
```

说明：

```text
左键：return true，让 item.shoot 发射 ShadowFlamePierceKnifeProjectile
右键：手动召回，return false，避免原版继续射出飞刀
```

### 5.3 OnHitNPC

生成驻留刀的位置按飞刀速度方向计算：

```csharp
Vector2 direction = Projectile.velocity.SafeNormalize(Vector2.UnitX);
float behindDistance = Math.Max(target.width, target.height) * 0.55f + 18f;
Vector2 stuckPosition = target.Center + direction * behindDistance;
```

命中逻辑：

```csharp
public override void OnHitNPC(NPC target, NPC.HitInfo hit, int damageDone)
{
    if (Projectile.owner == Main.myPlayer)
    {
        Vector2 direction = Projectile.velocity.SafeNormalize(Vector2.UnitX);
        Vector2 stuckPosition = target.Center + direction * (Math.Max(target.width, target.height) * 0.55f + 18f);

        ShadowFlameKnifeHelper.SpawnStuckKnife(
            Projectile.GetSource_FromAI(),
            stuckPosition,
            direction,
            Projectile.owner);
    }

    target.AddBuff(BuffID.ShadowFlame, 120);

    MeleeEffectAssets.NewProjectileDirect(
        Projectile.GetSource_FromAI(),
        target.Center,
        Vector2.Zero,
        ModContent.ProjectileType<SlashHitEffectProjectile>(),
        0,
        0f,
        Projectile.owner,
        Projectile.velocity.ToRotation());

    Projectile.Kill();
}
```

这里保留左键穿刺的基础暗影焰 Debuff，不把 Debuff 作为“四刀爆炸”的额外奖励。召回和爆炸首版可以不额外上 Debuff，避免伤害来源变复杂。

### 7.1 SpawnStuckKnife

生成前先扫描同玩家驻留刀：

```csharp
public static void SpawnStuckKnife(IEntitySource source, Vector2 position, Vector2 direction, int owner)
{
    KillOldestIfAtLimit(owner);

    Projectile projectile = Projectile.NewProjectileDirect(
        source,
        position,
        Vector2.Zero,
        ModContent.ProjectileType<ShadowFlameStuckKnifeProjectile>(),
        0,
        0f,
        owner);

    if (projectile.ModProjectile is ShadowFlameStuckKnifeProjectile stuck)
    {
        stuck.Initialize(direction);
    }
}
```

### 7.2 KillOldestIfAtLimit

```csharp
private static void KillOldestIfAtLimit(int owner)
{
    int count = 0;
    Projectile oldest = null;

    for (int i = 0; i < Main.maxProjectiles; i++)
    {
        Projectile p = Main.projectile[i];

        if (!p.active ||
            p.owner != owner ||
            p.type != ModContent.ProjectileType<ShadowFlameStuckKnifeProjectile>())
        {
            continue;
        }

        count++;

        if (oldest == null || p.timeLeft < oldest.timeLeft)
        {
            oldest = p;
        }
    }

    if (count >= ShadowFlameKnifeTuning.MaxStuckKnives && oldest != null)
    {
        oldest.Kill();
    }
}
```

这里用 `timeLeft` 判断最早生成的旧刀：驻留刀生成时都是 360 ticks，越早生成，剩余时间越少。

### 8.1 RecallAll

右键时扫描所有属于该玩家的驻留刀：

```csharp
public static void RecallAll(Player player, IEntitySource source, int baseDamage, float knockback)
{
    List<Projectile> stuckKnives = new();

    int stuckType = ModContent.ProjectileType<ShadowFlameStuckKnifeProjectile>();

    for (int i = 0; i < Main.maxProjectiles; i++)
    {
        Projectile p = Main.projectile[i];

        if (p.active && p.owner == player.whoAmI && p.type == stuckType)
        {
            stuckKnives.Add(p);
        }
    }

    if (stuckKnives.Count <= 0)
    {
        EmitEmptyRecallFeedback(player);
        return;
    }

    player.GetModPlayer<WeaponEffectsPlayer>().StartShadowFlameRecallSession();

    Vector2 recallTarget = player.Center;

    int recallDamage = Math.Max(1, (int)MathF.Round(baseDamage * ShadowFlameKnifeTuning.RecallDamageMultiplier));
    int explosionDamage = Math.Max(1, (int)MathF.Round(baseDamage * ShadowFlameKnifeTuning.ExplosionDamageMultiplier));

    foreach (Projectile stuck in stuckKnives)
    {
        ShadowFlameRecallKnifeProjectile.Spawn(
            source,
            stuck.Center,
            recallTarget,
            player.whoAmI,
            recallDamage,
            explosionDamage,
            knockback * 0.5f);

        stuck.Kill();
    }

    SoundStyle recallSound = new("WeaponEffects/Sounds/Slashing")
    {
        Volume = 0.55f,
        Pitch = -0.2f
    };

    MeleeEffectAssets.PlaySound(in recallSound, player.Center);
}
```

注意这里的 `recallTarget = player.Center` 是 **按下右键瞬间的玩家位置快照** 。召回飞刀不持续追踪玩家，这样玩家右键前的站位才有意义。

### 8.2 空召回反馈

产品说没有驻留刀时不造成伤害，只给轻反馈。实现上不要生成伤害 projectile。

```csharp
private static void EmitEmptyRecallFeedback(Player player)
{
    if (Main.dedServ)
    {
        return;
    }

    for (int i = 0; i < 3; i++)
    {
        Dust dust = Dust.NewDustDirect(
            player.position,
            player.width,
            player.height,
            ModContent.DustType<DarkSpark>(),
            Main.rand.NextFloat(-0.6f, 0.6f),
            Main.rand.NextFloat(-0.8f, 0.2f),
            0,
            new Color(80, 30, 150),
            0.45f);

        dust.noGravity = true;
    }
}
```

### 9.1 Spawn

```csharp
private int _explosionDamage;

public static void Spawn(
    IEntitySource source,
    Vector2 start,
    Vector2 target,
    int owner,
    int damage,
    int explosionDamage,
    float knockback)
{
    Vector2 direction = (target - start).SafeNormalize(Vector2.UnitX);
    Vector2 velocity = direction * ShadowFlameKnifeTuning.RecallSpeed;

    Projectile projectile = Projectile.NewProjectileDirect(
        source,
        start,
        velocity,
        ModContent.ProjectileType<ShadowFlameRecallKnifeProjectile>(),
        damage,
        knockback,
        owner,
        target.X,
        target.Y);

    if (projectile.ModProjectile is ShadowFlameRecallKnifeProjectile recall)
    {
        recall._explosionDamage = explosionDamage;
        projectile.netUpdate = true;
    }
}
```

注意：

```text

爆炸 projectile 不调用
```

这样可以保证爆炸条件只统计“本次召回中的不同飞刀”。

## 10. 四刀爆炸统计

修改：

```text
Common/Players/WeaponEffectsPlayer.cs
```

当前 `WeaponEffectsPlayer` 已经管理屏幕震动和连段状态，新增召回统计也适合放这里。项目架构也把玩家局部状态放在 `Common/Players`。([GitHub](https://raw.githubusercontent.com/nianba/WeaponEffects/master/ARCHITECTURE.md "raw.githubusercontent.com"))

### 10.1 新增字段

```csharp
private readonly int[] _shadowFlameRecallHitCount = new int[Main.maxNPCs];
private readonly int[] _shadowFlameRecallWindowTimer = new int[Main.maxNPCs];
private readonly bool[] _shadowFlameExplodedThisRecall = new bool[Main.maxNPCs];
```

### 10.2 StartShadowFlameRecallSession

```csharp
public void StartShadowFlameRecallSession()
{
    Array.Clear(_shadowFlameRecallHitCount);
    Array.Clear(_shadowFlameRecallWindowTimer);
    Array.Clear(_shadowFlameExplodedThisRecall);
}
```

每次右键召回开始时清空。这样能保证：

```text
每次召回独立统计
同一个敌人每次召回最多爆炸一次
不同敌人互不影响
```

### 10.3 RegisterShadowFlameRecallHit

```csharp
public void RegisterShadowFlameRecallHit(NPC target, Projectile recallProjectile, int explosionDamage)
{
    int npcIndex = target.whoAmI;

    if (npcIndex < 0 || npcIndex >= Main.maxNPCs)
    {
        return;
    }

    if (_shadowFlameRecallWindowTimer[npcIndex] <= 0)
    {
        _shadowFlameRecallHitCount[npcIndex] = 0;
    }

    _shadowFlameRecallHitCount[npcIndex]++;
    _shadowFlameRecallWindowTimer[npcIndex] = ShadowFlameKnifeTuning.ExplosionWindowTicks;

    if (_shadowFlameHitCount[npcIndex] < ShadowFlameKnifeTuning.ExplosionRequiredHits)
    {
        return;
    }

    if (_shadowFlameExplodedThisRecall[npcIndex])
    {
        return;
    }

    _shadowFlameExplodedThisRecall[npcIndex] = true;

    ShadowFlameExplosionProjectile.Spawn(
        recallProjectile.GetSource_FromAI(),
        target.Center,
        recallProjectile.owner,
        explosionDamage,
        recallProjectile.knockBack * 1.2f);
}
```

上面伪代码里变量名注意统一，实际应使用 `_shadowFlameRecallHitCount`。

### 10.4 PostUpdate 中衰减窗口

保留现有 `PostUpdate` 内容，在末尾追加：

```csharp
for (int i = 0; i < Main.maxNPCs; i++)
{
    if (_shadowFlameRecallWindowTimer[i] > 0)
    {
        _shadowFlameRecallWindowTimer[i]--;

        if (_shadowFlameRecallWindowTimer[i] <= 0)
        {
            _shadowFlameRecallHitCount[i] = 0;
        }
    }
}
```

这样“同时命中”不是绝对同一帧，而是约 12 ticks 的短窗口，符合产品要求的 0.15～0.25 秒容错。

## 12. 多人同步规则

首版不需要自定义 `ModPacket`，只使用 projectile 自带同步和 `SendExtraAI` / `ReceiveExtraAI`。

规则如下：

```text
手动生成多个 projectile 的逻辑，只在 Projectile.owner == Main.myPlayer 或 player.whoAmI == Main.myPlayer 执行
驻留刀 owner = player.whoAmI
召回刀 owner = player.whoAmI
爆炸 owner = player.whoAmI
召回计数存在每个玩家自己的 WeaponEffectsPlayer 中
不同玩家的驻留刀互不影响
不同玩家的召回计数互不影响
```

tModLoader 文档提醒，生成物品、生成 projectile、修改 tile 等逻辑通常应检查 `Projectile.owner == Main.myPlayer`，避免多人下重复执行。([tModLoader 文档](https://docs.tmodloader.net/docs/stable/class_mod_projectile.html "tModLoader: ModProjectile Class Reference"))

需要同步的字段：

```text
ShadowFlameRecallKnifeProjectile._explosionDamage
ShadowFlameStuckKnifeProjectile.rotation / 初始方向，如果不是直接写 Projectile.rotation
```

实现方式：

```csharp
public override void SendExtraAI(BinaryWriter writer)
{
    writer.Write(_explosionDamage);
}

public override void ReceiveExtraAI(BinaryReader reader)
{
    _explosionDamage = reader.ReadInt32();
}
```

## 14. 边界情况处理

### 14.1 没有驻留飞刀时右键

```text
不生成伤害 projectile
不清空任何状态
只生成 2～3 个很淡的 DarkSpark
```

### 14.2 左键击杀敌人

只要 `OnHitNPC` 触发，就生成驻留刀。不要检查 `target.active` 后再决定是否生成，否则击杀敌人时可能丢失布刀反馈。

### 14.3 驻留刀到期

```text
自然消失
不造成伤害
不触发召回
不触发爆炸
```

### 14.4 玩家死亡

驻留刀 `AI` 检查 owner：

```text
owner 不存在或 dead → timeLeft 压到 15 或直接 Kill
```

召回飞刀可以自然飞完，不需要额外处理。

### 14.5 切换武器

驻留刀 `AI` 检查：

```text
owner.HeldItem.type != ItemID.ShadowFlameKnife → timeLeft 压到 15
```

这样玩家切武器后无法长期保留飞刀资源。

### 14.6 超过 12 把驻留刀

```text
新刀正常生成
最早生成的旧刀 Kill
旧刀不造成伤害
旧刀不触发爆炸
```

### 14.7 召回飞刀穿墙

```text
ShadowFlameRecallKnifeProjectile.tileCollide = false
```

左键飞刀仍然：

```text
ShadowFlamePierceKnifeProjectile.tileCollide = true
```

## 15. 开发顺序

```text
1. 在 SlashGlobalItem.ShouldUseSlashAction 排除 ItemID.ShadowFlameKnife

```
