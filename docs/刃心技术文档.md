
# 刃心技术方案

## 1. 实现目标

本次实现 11 个饰品：

```text
刃势线：
疾风剑符
破甲剑符
锐利剑符
三刃势徽章

重斩线：
断岳坠饰
延锋坠饰
崩锋徽章

蓄力线：
凝势沙漏
长锋晶坠
裂势印记
聚势纹章

最终：
刃心
```

最终机制收敛为：

```text
刃势线：直接提供近战速度、暴击、近战伤害
重斩线：强化第四段 combo
蓄力线：强化右键蓄力斩
刃心：整合三条线最终饰品能力
```

# 2. 刃势线实现

## 2.1 刃势线不再进入 WeaponEffectsPlayer

刃势线三个饰品直接在 `UpdateAccessory` 里改玩家近战属性。

tModLoader 的 `ModItem.UpdateAccessory(Player, bool)` 就是饰品效果入口，适合直接给玩家加属性；`Player.GetAttackSpeed(DamageClass.Melee)`、`Player.GetCritChance(DamageClass.Melee)`、`Player.GetDamage(DamageClass.Melee)` 分别对应近战速度、暴击率和伤害修改。([docs.tmodloader.net](https://docs.tmodloader.net/docs/stable/class_mod_item.html "tModLoader: ModItem Class Reference")) ([docs.tmodloader.net](https://docs.tmodloader.net/docs/stable/class_player.html "tModLoader: Player Class Reference"))

## 2.2 数值保持文档原数值

原文档中疾风剑符满层为 +16% 攻速，破甲剑符满层为 +8% 暴击，锐利剑符满层为 +10% 伤害。现在去掉层数系统后，直接把这些满额效果作为常驻效果：

```text
疾风剑符：+16% 近战速度
破甲剑符：+8% 近战暴击率
锐利剑符：+10% 近战伤害
三刃势徽章：综合三者
刃心：继承三刃势徽章效果
```

实现示例：

```csharp
public override void UpdateAccessory(Player player, bool hideVisual)
{
    player.GetAttackSpeed(DamageClass.Melee) += 0.16f;
}
```

```csharp
public override void UpdateAccessory(Player player, bool hideVisual)
{
    player.GetCritChance(DamageClass.Melee) += 8f;
}
```

```csharp
public override void UpdateAccessory(Player player, bool hideVisual)
{
    player.GetDamage(DamageClass.Melee) += 0.10f;
}
```

三刃势徽章：

```csharp
public override void UpdateAccessory(Player player, bool hideVisual)
{
    player.GetAttackSpeed(DamageClass.Melee) += 0.16f;
    player.GetCritChance(DamageClass.Melee) += 8f;
    player.GetDamage(DamageClass.Melee) += 0.10f;
}
```

# 3. 近战速度与自定义剑气的兼容

这里需要一个小改动。

当前 `SlashChannelProjectile.NormalSlashInterval` 只根据 `_useAnimation` 和配置里的 `NormalSlashIntervalMultiplier` 计算普通剑气间隔，没有读取玩家当前近战速度。也就是说，如果只在饰品里写 `player.GetAttackSpeed(DamageClass.Melee) += 0.16f`，它会影响原版近战速度，但不一定影响 WeaponEffects 的自定义剑气频率。当前代码里普通攻击间隔就是在 `NormalSlashInterval` 里直接返回 `_useAnimation * multiplier`。([GitHub](https://github.com/nianba/WeaponEffects/blob/master/Content/Projectiles/SlashChannelProjectile.cs "WeaponEffects/Content/Projectiles/SlashChannelProjectile.cs at master · nianba/WeaponEffects · GitHub"))

因此需要把 `NormalSlashInterval` 改成读取近战速度：

```csharp
private int NormalSlashInterval
{
    get
    {
        float multiplier = MathHelper.Clamp(
            ModContent.GetInstance<WeaponEffectsGameplayConfig>().NormalSlashIntervalMultiplier,
            0.25f,
            3f
        );

        float interval = Math.Max(1, _useAnimation) * multiplier;

        if (Projectile.owner >= 0 && Projectile.owner < Main.maxPlayers)
        {
            Player player = Main.player[Projectile.owner];

            if (player.active)
            {
                float meleeSpeed = MathHelper.Clamp(
                    player.GetAttackSpeed(DamageClass.Melee),
                    0.25f,
                    4f
                );

                interval /= meleeSpeed;
            }
        }

        return Math.Max(1, (int)MathF.Round(interval));
    }
}
```

这个改动很小，但很重要。它可以让：

```text
疾风剑符
三刃势徽章
刃心
原版近战速度饰品
```

都正确影响 WeaponEffects 的普通剑气频率。

# 4. 重斩线实现

重斩线只需要改 `SlashChannelProjectile.FireComboSchemeASlash()`。

当前普通攻击 combo 会通过：

```csharp
int comboStepIndex = player.GetModPlayer<WeaponEffectsPlayer>().ConsumeNextSlashComboStep();
ref readonly SlashComboStep step = ref Compact3DComboSchemeA.GetStep(comboStepIndex);
```

取得当前 combo 段数。`WeaponEffectsPlayer.ConsumeNextSlashComboStep()` 会推进 `SlashComboStepIndex`，并按 `Compact3DComboSchemeA.Count` 循环，所以第四段可以直接判断 `comboStepIndex == 3`。([GitHub](https://github.com/nianba/WeaponEffects/blob/master/Common/Players/WeaponEffectsPlayer.cs "WeaponEffects/Common/Players/WeaponEffectsPlayer.cs at master · nianba/WeaponEffects · GitHub")) 当前 `FireComboSchemeASlash()` 已经在创建剑气前计算了 `length`、`thicknessScale`、`yScale`，并把 `NormalSlashDamage` 传给 `CreateProfiledSlash`。([GitHub](https://github.com/nianba/WeaponEffects/blob/master/Content/Projectiles/SlashChannelProjectile.cs "WeaponEffects/Content/Projectiles/SlashChannelProjectile.cs at master · nianba/WeaponEffects · GitHub"))

## 4.1 在 WeaponEffectsPlayer 增加两个字段

```csharp
public float FourthSlashDamageMultiplier = 1f;
public float FourthSlashLengthMultiplier = 1f;
```

增加注册方法：

```csharp
public void RegisterHeavySlash(float damageMultiplier, float lengthMultiplier)
{
    FourthSlashDamageMultiplier = Math.Max(FourthSlashDamageMultiplier, damageMultiplier);
    FourthSlashLengthMultiplier = Math.Max(FourthSlashLengthMultiplier, lengthMultiplier);
}
```

在 `ResetEffects()` 中重置：

```csharp
public override void ResetEffects()
{
    FourthSlashDamageMultiplier = 1f;
    FourthSlashLengthMultiplier = 1f;

    ChargeReadyFrameOffset = 0;
    ChargeLengthBonusAtHalf = 0f;
    ChargeLengthBonusAtFull = 0f;
    ChargeDamageBonusAtHalf = 0f;
    ChargeDamageBonusAtFull = 0f;
    ChargeWidthBonusAtFull = 0f;
}
```

`ResetEffects()` 正是 tModLoader 推荐用于重置 `ModPlayer` 条件字段的位置。([docs.tmodloader.net](https://docs.tmodloader.net/docs/stable/class_mod_player.html "tModLoader: ModPlayer Class Reference"))

## 4.2 重斩线数值

```text
断岳坠饰：
第四段 combo 伤害 +30%

延锋坠饰：
第四段 combo 距离 +55%

崩锋徽章：
第四段 combo 伤害 +35%
第四段 combo 距离 +55%
```

饰品写法：

```csharp

```

```csharp
// 延锋坠饰

    .RegisterHeavySlash(1.00f, 1.55f);
```

```csharp

    .RegisterHeavySlash(1.35f, 1.55f);
```

## 4.3 修改第四段 combo

在 `FireComboSchemeASlash()` 里：

```csharp
int comboStepIndex = player.GetModPlayer<WeaponEffectsPlayer>().ConsumeNextSlashComboStep();
ref readonly SlashComboStep step = ref Compact3DComboSchemeA.GetStep(comboStepIndex);

var effectsPlayer = player.GetModPlayer<WeaponEffectsPlayer>();

float length = _weaponLength * LegacyAverageLengthScale * step.LengthScale;
float thicknessScale = step.ThicknessScale;

int damage = NormalSlashDamage;

if (comboStepIndex == 3)
{
    length *= effectsPlayer.FourthSlashLengthMultiplier;
    damage = Math.Max(1, (int)MathF.Round(damage * effectsPlayer.FourthSlashDamageMultiplier));
}
```

然后把 `CreateProfiledSlash` 里的：

```csharp
damage: NormalSlashDamage,
```

改成：

```csharp
damage: damage,
```

第一版只改第四斩的伤害和距离，不新增特殊视觉字段。这样不需要改 `SlashArcProjectile.SendExtraAI / ReceiveExtraAI`，多人同步风险最低。你的架构文档也提醒过，新增投射物字段需要同步，否则多人下可能出错。([GitHub](https://github.com/nianba/WeaponEffects/blob/master/ARCHITECTURE.md "WeaponEffects/ARCHITECTURE.md at master · nianba/WeaponEffects · GitHub"))

# 5. 蓄力线实现

蓄力线只改 `ChargedSlashProjectile`。

当前蓄力斩已有：

```csharp
MinChargeFrame
ChargeReadyFrame
ChargeProgress
CurrentLengthScale
CurrentDamageScale
ReleaseChargedSlash()
```

蓄力释放时会在 `ReleaseChargedSlash()` 中计算 `lengthScale` 和 `damage`，然后创建 `SlashArcProjectile`。([GitHub](https://github.com/nianba/WeaponEffects/blob/master/Content/Projectiles/ChargedSlashProjectile.cs "WeaponEffects/Content/Projectiles/ChargedSlashProjectile.cs at master · nianba/WeaponEffects · GitHub")) ([GitHub](https://github.com/nianba/WeaponEffects/blob/master/Content/Projectiles/ChargedSlashProjectile.cs "WeaponEffects/Content/Projectiles/ChargedSlashProjectile.cs at master · nianba/WeaponEffects · GitHub"))

## 5.1 在 WeaponEffectsPlayer 增加蓄力字段

```csharp
public int ChargeReadyFrameOffset;

public float ChargeLengthBonusAtHalf;
public float ChargeLengthBonusAtFull;

public float ChargeDamageBonusAtHalf;
public float ChargeDamageBonusAtFull;

public float ChargeWidthBonusAtFull;
```

注册方法：

```csharp
public void RegisterChargeAccessory(
    int readyFrameOffset,
    float lengthBonusAtHalf,
    float lengthBonusAtFull,
    float damageBonusAtHalf,
    float damageBonusAtFull,
    float widthBonusAtFull)
{
    ChargeReadyFrameOffset = Math.Min(ChargeReadyFrameOffset, readyFrameOffset);

    ChargeLengthBonusAtHalf = Math.Max(ChargeLengthBonusAtHalf, lengthBonusAtHalf);
    ChargeLengthBonusAtFull = Math.Max(ChargeLengthBonusAtFull, lengthBonusAtFull);

    ChargeDamageBonusAtHalf = Math.Max(ChargeDamageBonusAtHalf, damageBonusAtHalf);
    ChargeDamageBonusAtFull = Math.Max(ChargeDamageBonusAtFull, damageBonusAtFull);

    ChargeWidthBonusAtFull = Math.Max(ChargeWidthBonusAtFull, widthBonusAtFull);
}
```

## 5.3 修改蓄满时间

当前 `ChargeReadyFrame` 是根据 `MinChargeFrame` 和配置倍率算出来的。([GitHub](https://github.com/nianba/WeaponEffects/blob/master/Content/Projectiles/ChargedSlashProjectile.cs "WeaponEffects/Content/Projectiles/ChargedSlashProjectile.cs at master · nianba/WeaponEffects · GitHub"))

新增：

```csharp
private int EffectiveChargeReadyFrame
{
    get
    {
        var effectsPlayer = Main.player[Projectile.owner].GetModPlayer<WeaponEffectsPlayer>();
        return Math.Max(MinChargeFrame, ChargeReadyFrame + effectsPlayer.ChargeReadyFrameOffset);
    }
}
```

然后把：

```csharp
private float ChargeProgress => MathHelper.Clamp(Projectile.ai[1] / ChargeReadyFrame, 0f, 1f);
```

改成：

```csharp
private float ChargeProgress => MathHelper.Clamp(Projectile.ai[1] / EffectiveChargeReadyFrame, 0f, 1f);
```

并把蓄满视觉判断里的：

```csharp
Projectile.ai[1] >= ChargeReadyFrame
```

改成：

```csharp
Projectile.ai[1] >= EffectiveChargeReadyFrame
```

`DrawChargeBar()` 里的进度也使用 `EffectiveChargeReadyFrame`，否则凝势沙漏缩短蓄力后，UI 进度会和实际蓄力不一致。

## 5.4 修改蓄力斩距离、伤害、宽度

加一个简单插值方法：

```csharp
private static float EvaluateHalfFullBonus(float progress, float halfBonus, float fullBonus)
{
    progress = MathHelper.Clamp(progress, 0f, 1f);

    if (progress <= 0.5f)
    {
        return MathHelper.Lerp(0f, halfBonus, progress / 0.5f);
    }

    return MathHelper.Lerp(halfBonus, fullBonus, (progress - 0.5f) / 0.5f);
}
```

在 `ReleaseChargedSlash()` 里，当前代码已经有：

```csharp
float lengthScale = CurrentLengthScale;
int damage = Math.Max(1, (int)MathF.Round(Projectile.damage * CurrentDamageScale));
```

改成：

```csharp
var effectsPlayer = player.GetModPlayer<WeaponEffectsPlayer>();

float progress = ChargeProgress;

float lengthBonus = EvaluateHalfFullBonus(
    progress,
    effectsPlayer.ChargeLengthBonusAtHalf,
    effectsPlayer.ChargeLengthBonusAtFull
);

float damageBonus = EvaluateHalfFullBonus(
    progress,
    effectsPlayer.ChargeDamageBonusAtHalf,
    effectsPlayer.ChargeDamageBonusAtFull
);

float widthBonus = effectsPlayer.ChargeWidthBonusAtFull * progress;

float lengthScale = CurrentLengthScale * (1f + lengthBonus);

int damage = Math.Max(
    1,
    (int)MathF.Round(Projectile.damage * CurrentDamageScale * (1f + damageBonus))
);

float thickness = 0.45f * (1f + widthBonus);
```

然后把 `CreateSlash` 里的：

```csharp
thickness: 0.45f,
```

改成：

```csharp
thickness: thickness,
```

注意：裂势印记要求“蓄力伤害提升在当前蓄力最终伤害基础上做乘法”。上面这行：

```csharp
Projectile.damage * CurrentDamageScale * (1f + damageBonus)
```

正好满足这个要求。

```text

```

```csharp
public override void UpdateAccessory(Player player, bool hideVisual)
{
    // 刃势线：常驻近战属性

        -90,
        0.10f,
        0.25f,
        0.10f,
        0.25f,
        0.10f
    );
}
```

# 8. 饰品冲突规则

第一版可以先不做复杂冲突系统，因为最简单可用。

但我建议至少加一个轻量规则：

```text
装备刃心后，不允许再装备：
三刃势徽章
崩锋徽章
聚势纹章
以及三条线的小组件
```

原因是刃心已经包含全部能力，如果允许继续叠加子组件，强度会膨胀。

如果要简化实现，第一版也可以只靠 `RegisterHeavySlash` / `RegisterChargeAccessory` 的 `Math.Max` 防止重斩和蓄力重复叠加；刃势线是原版属性，重复装备会真的叠加，所以最好至少给刃势线做冲突，或者在 tooltip 里先声明“不建议与上级合成饰品同时装备”。

最稳方案：

```text
三刃势徽章不能和疾风剑符 / 破甲剑符 / 锐利剑符同时装备
崩锋徽章不能和断岳坠饰 / 延锋坠饰同时装备
聚势纹章不能和凝势沙漏 / 长锋晶坠 / 裂势印记同时装备
刃心不能和三条线任意子饰品同时装备
```

# 11. 最终验证清单

```text
无饰品：
原有普通攻击、第四段 combo、右键蓄力表现不变。

疾风剑符：
WeaponEffects 普通剑气频率变快，数值为 +16% 近战速度。

破甲剑符：
近战暴击率增加 8%。

锐利剑符：
近战伤害增加 10%。

三刃势徽章：
同时获得 +16% 近战速度、+8% 暴击、+10% 近战伤害。

断岳坠饰：
只有第四段 combo 伤害 +30%。

延锋坠饰：
只有第四段 combo 距离 +55%。

崩锋徽章：
第四段 combo 伤害 +35%，距离 +55%。

凝势沙漏：
满蓄时间 -1.5 秒，蓄力 UI 正确。

长锋晶坠：
半蓄距离 +10%，满蓄距离 +25%，剑气宽度略增。

裂势印记：
半蓄伤害 +10%，满蓄伤害 +25%，且是在当前蓄力最终伤害基础上乘法增加。

聚势纹章：
综合三件蓄力饰品能力。

刃心：
综合三刃势徽章、崩锋徽章、聚势纹章能力。
```

构建命令继续用：

```powershell
dotnet build .\WeaponEffects.csproj
```

你的架构文档也建议每次改动后先构建，再测试普通剑、发射弹幕的剑、蓄力开关、多人同步和特殊武器兼容。([GitHub](https://github.com/nianba/WeaponEffects/blob/master/ARCHITECTURE.md "WeaponEffects/ARCHITECTURE.md at master · nianba/WeaponEffects · GitHub"))

## 最终落地结论

这版最终方案可以概括为：

> **刃势线直接做常驻近战属性；重斩线只改第四段 combo 的伤害和距离；蓄力线只改右键蓄力斩的蓄满时间、距离、宽度和伤害；刃心只做三条线能力整合。**

这是当前最稳、最容易实现、最不容易破坏现有剑气系统的版本。
